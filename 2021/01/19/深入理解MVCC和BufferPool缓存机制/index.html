深入理解MVCC和BufferPool缓存机制

发表于 2021-01-19 阅读次数： Valine：
mysql的隔离级别
  • 读未提交：一个事务在未提交时，他所作的变更被其他事务查询到。
  • 读提交：一个事务提交之后，他所作的变更被其他事物查询到。
  • 可重复读：一个事务在执行过程中看到的数据，总是跟这个事务刚启动时看到的数据是一致的。
  • 串行化：对于同一行记录，读会加读锁，写会加写锁，将所有并发执行的事务串行化。
MVCC机制

多版本并发控制：mysql在可重复读隔离级别下保证隔离性就是通过MVCC(multi-version concurrency control)机制来实现的。

  • undo log：回滚日志，mysql在更新一条记录时，会将原数据记录到undo log中，事务回滚时，读取undo log中的数据，实现回滚。
BUFFER POOL

一条更新语句执行流程：update t set name =’lulu’ where id =2;

server层略过

  1. 先将id为2的行取出，放到Buffer Pool中
  2. 将原始值记录到undo log
  3. 更新Buffer Pool
  4. 写 redo log Buffer。
  5. 准备提交事务。
  6. 写redo log prepared
  7. 写bin log
  8. 写一个commit的标记到redo log
  9. 事务提交
